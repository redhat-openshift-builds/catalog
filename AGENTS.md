# OpenShift Builds FBC Catalog

File-Based Catalog (FBC) repo for the OpenShift Builds operator. Generates OLM catalog images for each supported OpenShift version.

## Architecture

- `config.yaml` — central source of truth. Defines the OCP support matrix, bundle digests, channel config, and deprecation settings.
- `fbc/` — generated FBC catalog files, one subdirectory per OCP version (4.12–4.21). Never hand-edit — regenerated by `make generate`.
- `scripts/` — generation and install scripts (`generate.sh`, `install.sh`, helpers)
- `bin/` — locally installed tools (`opm`, `yq`, `kustomize`) — gitignored
- `.tekton/` — Konflux pipeline definitions, one PR + push pair per OCP version

This is a pure config/generation repo — no Go code, no application logic. The only "source" is `config.yaml` and the scripts. Everything under `fbc/` is generated output.

## Build & Test Commands

- Install tools: `make install` (installs `opm`, `yq`, `kustomize` into `bin/`)
- Generate all catalogs: `make generate`
- Generate for specific OCP: `make generate OCP=4.17`
- Add specific bundle: `make generate BUNDLE="registry.redhat.io/...@sha256:..."`
- Rebuild from scratch: `make generate OCP=4.17 REBUILD="true"`

No tests or lint — validation is done by Konflux pipelines on PR.

## Key Conventions

- **`config.yaml` is the source of truth** — all changes start here. `fbc/` files are regenerated from it.
- **Never hand-edit `fbc/` files** — always modify `config.yaml` then run `make generate`.
- **Bundle insertion order matters** — new entries go after the last entry of the same major.minor series, not appended globally.
- **Bundle digest format** — use `registry.redhat.io` format, not Konflux `quay.io` format. Convert when extracting from Konflux snapshots.
- **PATH setup required** — run `PATH="$PWD/bin:$PATH"` before `make generate` so locally installed tools are found.

## PR Conventions

- **Commit flags:** always use `-s` (sign-off) and `-S` (GPG sign)
- **Branch naming:** `builds-VERSION` (e.g., `builds-1.8.0`, `builds-1.6.5-1.7.2` for multi-version)
- **Commit message:** `Add catalog entries for VERSION_LIST`
- **PR title:** matches commit message
- **PR body:** `# Changes:` header with bullet points per version, plus OCP support matrix updates for minor releases
- **Fork-aware push:** detect fork vs upstream for `gh pr create --head` flag
- **One commit per PR:** amend and force push — no references to incremental changes
